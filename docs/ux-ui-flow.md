# Smart Farm - UX/UI Flow for Line OA + LIFF

บทความนี้อธิบายรายละเอียด User Experience และ User Interface Flow สำหรับการใช้งานผ่าน Line Official Account และ LIFF (Line Front-end Framework)

---

## Rich Menu Design

### Layout

```
┌──────────────────────┬──────────────────────┐
│   📍 แปลงของฉัน       │   🌱 เริ่มปลูก        │
│   My Plots           │   Start Planting     │
├──────────────────────┼──────────────────────┤
│   📝 บันทึกวันนี้      │   📊 สรุปรายได้       │
│   Log Activity       │   Dashboard          │
├──────────────────────┴──────────────────────┤
│         👤 โปรไฟล์ของฉัน                      │
│         My Profile                          │
└─────────────────────────────────────────────┘
```

### Menu Item Descriptions

| Menu Item | Action | Opens |
|-----------|--------|-------|
| 📍 แปลงของฉัน | เปิด LIFF | หน้ารายการแปลง (Plot List) |
| 🌱 เริ่มปลูก | เปิด LIFF | หน้าเลือกแปลงและเริ่มรอบใหม่ |
| 📝 บันทึกวันนี้ | เปิด LIFF | หน้าบันทึกกิจกรรม (Activity Form) |
| 📊 สรุปรายได้ | เปิด LIFF | Dashboard สรุปรายได้-ค่าใช้จ่าย |
| 👤 โปรไฟล์ | เปิด LIFF | หน้าโปรไฟล์และการตั้งค่า |

## User Flow 1: First-Time User (การใช้งานครั้งแรก)

```mermaid
flowchart TD
    A[User เพิ่ม Line OA<br/>Smart Farm] --> B[ส่งข้อความต้อนรับ]
    B --> C{Line Login<br/>Authorization}
    C -->|Allow| D[Backend:<br/>สร้าง User + OAuth Record]
    C -->|Deny| E[แสดงข้อความ<br/>กรุณาอนุญาตเพื่อใช้งาน]
    D --> F[แสดง Rich Menu]
    F --> G[ส่งข้อความแนะนำ:<br/>เริ่มต้นด้วยการเพิ่มแปลง]
    G --> H[User กด 📍 แปลงของฉัน]
    H --> I[เปิด LIFF:<br/>แสดงหน้าว่าง No Plots]
    I --> J[แสดงปุ่ม<br/>+ เพิ่มแปลงแรก]
    J --> K[User Flow 2:<br/>เพิ่มแปลง]
```

**Welcome Message Example**:
```
🌾 ยินดีต้อนรับสู่ Smart Farm! 🌾

ระบบบริหารจัดการแปลงเกษตรของคุณ

เริ่มต้นง่ายๆ เพียง 3 ขั้นตอน:
1️⃣ เพิ่มแปลงเกษตร
2️⃣ เริ่มรอบการปลูก
3️⃣ บันทึกกิจกรรมรายวัน

กดเมนูด้านล่างเพื่อเริ่มต้น!
```

## User Flow 2: เพิ่มแปลงเกษตร

```mermaid
flowchart TD
    A[User กด<br/>📍 แปลงของฉัน] --> B[เปิด LIFF:<br/>Plot List Page]
    B --> C{มีแปลงหรือไม่?}
    C -->|ไม่มี| D[แสดงหน้าว่าง<br/>Empty State]
    C -->|มี| E[แสดง Plot Cards]
    D --> F[ปุ่ม + เพิ่มแปลงแรก]
    E --> F[ปุ่ม + เพิ่มแปลง FAB]
    F --> G[เปิด Modal/Page:<br/>Add Plot Form]
    G --> H[กรอกข้อมูล:<br/>1. ชื่อแปลง]
    H --> I[2. ขนาดพื้นที่<br/>เลือก: ไร่/งาน/วา หรือ ตร.ม.]
    I --> J[3. ข้อมูลดิน Optional<br/>pH, ประเภทดิน]
    J --> K[4. เลือกตำแหน่งบนแผนที่]
    K --> L[User กด<br/>เปิด Google Maps]
    L --> M[แสดง Google Maps<br/>พร้อม Current Location]
    M --> N{User เลือกตำแหน่ง}
    N -->|Drop Pin| O[ยืนยันตำแหน่ง]
    N -->|ใช้ตำแหน่งปัจจุบัน| O
    O --> P[กลับมาฟอร์ม<br/>แสดง Lat/Lng]
    P --> Q[User กด บันทึก]
    Q --> R[Validate Form]
    R -->|Valid| S[POST /api/v1/plots]
    R -->|Invalid| T[แสดง Error Messages]
    T --> H
    S --> U[แสดง Toast:<br/>เพิ่มแปลงสำเร็จ]
    U --> V[Refresh Plot List<br/>แสดง Card ใหม่]
```

## User Flow 3: เริ่มรอบการปลูก

```mermaid
flowchart TD
    A[User กด<br/>🌱 เริ่มปลูก] --> B{มีแปลงหรือไม่?}
    B -->|ไม่มี| C[แสดงข้อความ:<br/>กรุณาเพิ่มแปลงก่อน]
    C --> D[ปุ่ม: เพิ่มแปลงเลย]
    D --> E[User Flow 2]
    B -->|มี| F[เปิด LIFF:<br/>Select Plot Page]
    F --> G[แสดงรายการแปลง<br/>พร้อมสถานะ]
    G --> H[User เลือกแปลง]
    H --> I{แปลงมี Cycle Active?}
    I -->|มี| J[แสดง Warning:<br/>มีการปลูกอยู่แล้ว]
    J --> K[ตัวเลือก:<br/>1. ดูรายละเอียด<br/>2. เลือกแปลงอื่น]
    I -->|ไม่มี| L[เปิดหน้า:<br/>Select Crop Type]
    L --> M[GET /public/crop-types<br/>แสดงประเภทพืช]
    M --> N[User เลือกประเภท<br/>เช่น ผลไม้]
    N --> O[GET /public/crop-varieties<br/>?crop_type_id=1]
    O --> P[แสดงสายพันธุ์<br/>พร้อมรูปภาพ]
    P --> Q[User เลือกพันธุ์<br/>เช่น มะม่วงน้ำดอกไม้]
    Q --> R[GET /public/standard-plans/:variety_id]
    R --> S{มี Template?}
    S -->|มี| T[แสดง Template Preview<br/>Timeline + ค่าใช้จ่ายประมาณ]
    S -->|ไม่มี| U[ข้อความ: ไม่มีแผนมาตรฐาน<br/>กรอกวันที่เริ่มเอง]
    T --> V[User เลือกวันเริ่มปลูก<br/>Date Picker]
    U --> V
    V --> W[แสดง Expected Harvest Date<br/>คำนวณจาก total_days]
    W --> X[กรอกชื่อรอบ Optional<br/>เช่น มะม่วงรุ่น 1/2567]
    X --> Y[User กด เริ่มปลูก]
    Y --> Z[POST /api/v1/cycles]
    Z --> AA[แสดง Success Message<br/>พร้อม Timeline]
    AA --> AB[Navigate to:<br/>Cycle Detail Page]
```

## User Flow 5: ดู Dashboard

```mermaid
flowchart TD
    A[User กด<br/>📊 สรุปรายได้] --> B[เปิด LIFF:<br/>Dashboard Page]
    B --> C[GET /plots<br/>GET /cycles]
    C --> D[แสดงสรุปภาพรวม:<br/>1. รายได้รวม<br/>2. ค่าใช้จ่ายรวม<br/>3. กำไร/ขาดทุน]
    D --> E[Chart: รายได้แต่ละแปลง<br/>Pie Chart]
    E --> F[Chart: รายได้-ค่าใช้จ่าย<br/>Bar Chart รายเดือน]
```

## Accessibility Considerations

1. **Font Size**: ขนาดตัวอักษรอย่างน้อย 16px
2. **Touch Targets**: ปุ่มขนาดอย่างน้อย 44x44px
3. **Color Contrast**: WCAG AA compliance
4. **Loading States**: แสดง Skeleton/Spinner เมื่อโหลดข้อมูล
5. **Error Messages**: ข้อความชัดเจน เป็นภาษาไทย

---

## Security Best Practices

1. **JWT Validation**: ตรวจสอบ Token ทุก API Request
2. **Input Sanitization**: ป้องกัน XSS, SQL Injection
3. **File Upload Validation**: ตรวจสอบ MIME Type และขนาดไฟล์
4. **HTTPS Only**: ไม่อนุญาตให้ใช้ HTTP
5. **CORS Policy**: จำกัด Origin ที่อนุญาต
